# Disabled Tests Policy

All disabled/skipped tests must be documented here with an associated issue tracker link.

## Rules
1. Every disabled test must have an open issue tracking the reason for disablement
2. Commented-out tests must be listed here with rationale
3. Stale entries (issues closed, tests re-enabled) must be removed promptly
4. CI will fail if a new `DISABLED`/`SKIP` annotation is added without a corresponding entry here
